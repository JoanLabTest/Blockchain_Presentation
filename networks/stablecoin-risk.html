Variables et contraintes algorithmiques utilisées par le Risk Engine pour simuler les chocs systémiques sur les infrastructures de règlement privées (USDT/USDC).

Depeg Stress Test

Modélisation de la déviation du prix secondaire par rapport à la valeur nominale fiat. Évalue la rapidité de l'ancrage à se briser sous la pression vendeuse.

Tolerance Threshold ± 0.50%
Volatility Trigger > 100bps/hr
// Risk Engine Logic if (secondaryMarketPrice < 0.995) { triggerAlert("Depeg Risk [Red]"); initiateArbSimulation(liquidityPools); }

Liquidity Run Simulation

Simulation d'un "Bank Run". Modélise la capacité de l'émetteur à honorer des retraits massifs (combustion) en liquidant ses réserves fiat.

Reserve Baseline $100B
Residual Liquidity $55B

Redemption Queue Modeling

Modélise les frictions opérationnelles lorsque les demandes fiat (burn) dépassent la capacité de traitement bancaire de l'émetteur, créant un carnet de commandes en souffrance.

Max Processing Rate $2B / Day
Slippage Penalty T+2 delays
// Queue Simulator const backlog = redemptionReqs - processingCap; if (backlog > 0) { extrapolateDepeg(backlog / totalReserves); }

Reserve Duration Mismatch

Modélise le risque de taux d'intérêt de l'émetteur. Si les réserves sont investies dans des bons du Trésor à long terme, une hausse des taux crée une perte latente (Mark-to-Market).

Weighted Avg Maturity < 90 Days
Yield Sensitivity +100bps = -0.2%
