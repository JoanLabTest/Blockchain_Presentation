Architecture de la Spécification

Le TFIN ID suit une structure déterministe qui encode les caractéristiques primaires de l'actif directement au sein de l'identifiant. Ce format, lisible par l'homme mais analysable par machine, garantit l'interopérabilité à travers la pile institutionnelle à 5 couches.

TFIN - DEBT - ETH - 2026 - 0001
Classe d'Actif Code de 4 lettres mapping vers le standard de classification institutionnel TFIC (DEBT, EQTY, FUND, DRVT).
Type de Registre Code de 3 lettres identifiant l'infrastructure DLT sous-jacente (Public : ETH, POL | Privé : HYP, CRD).
Registre Institutionnel Tous les TFIN émis sont répertoriés dans le Global Tokenized Securities Registry (GTSR) pour une traçabilité multi-chaînes.
Définition de Schéma JSON (tfin-schema.json) 
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "title": "TokenizedFinancialInstrumentNumber",
  "type": "object",
  "properties": {
    "tfinId": {
      "type": "string",
      "pattern": "^TFIN-[A-Z]{4}-[A-Z]{3}-[0-9]{4}-[0-9]{4}$"
    },
    "classification": {
      "type": "string",
      "enum": ["DEBT", "EQTY", "FUND", "STRU"]
    },
    "ledger": {
      "type": "string",
      "enum": ["ETH", "POL", "STR", "HYP"]
    }
  },
  "required": ["tfinId", "classification", "ledger"]
}
Avertissement Technique : La syntaxe regex est optimisée pour les moteurs RegExp ECMAScript. Certains environnements DLT ou bases de données SQL peuvent exiger des ajustements de paramètres d'échappement (ex: omission des marqueurs de limites ^ et $ dans les schémas PostgreSQL spécifiques). La synchronisation avec des registres hors ligne est soumise à des latences de mise à jour.
