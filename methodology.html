Méthodologie de Recherche

Documentation technique des modèles, sources de données et algorithmes utilisés sur cette plateforme.

1. Sources de Données

DCM Digital Hub agrège des données provenant de sources primaires et secondaires pour garantir l'exactitude des analyses.

Flux de Marché (Market Data)

  • Prix Actifs (Crypto/Fiats) : CoinGecko API (Pro Plan) & Chainlink Price Feeds (On-chain Oracle).
  • Rendements Staking : Beacon Chain Direct Data (Ethereum) & Validator Performance Indices.
  • Risk Free Rate : US Treasury 10Y Yield (Bloomberg / FRED Data).

Données On-Chain

  • Token Standards : ERC-3643 Association & EIP Repository.
  • Transaction Volume : Etherscan & Dune Analytics Public Dashboards.

2. Algorithme de Recherche (Search 4.0)

Le moteur de recherche interne n'utilise pas de base de données SQL classique mais un index inversé vectoriel généré statiquement.

Calcul du Score de Pertinence

Le classement des résultats est déterminé par un algorithme TF-IDF pondéré par le contexte :

Score = (TF × IDF) × (Field_Weight) × (Context_Boost)
// PONDÉRATION DES CHAMPS (Field_Weight) const WEIGHTS = { title: 10, // Correspondance exacte dans le titre h2: 5, // Sous-titre keywords: 8, // Mots-clés méta content: 1 // Corps du texte };

Un "Context Boost" de x1.5 est appliqué si le terme recherché appartient à la même catégorie sémantique que la page en cours de lecture (ex: Recherche "Yield" depuis la page "Validator Economics").

3. Modèles Financiers

Calcul du ROI Validateur

Le simulateur présent sur pos-economics.html utilise la formule suivante pour estimer le rendement net :

ROI_Net = [ (Stake × APY_Gross) - OpEx_Yearly ] / Stake

Courbe de Dilution du Rendement

La projection de baisse du rendement (Yield Decay) suit une courbe logarithmique inverse basée sur le taux d'émission du protocole Ethereum (Issuance Curve) :

Yield(y) = c / sqrt(Total_Staked) // Où 'c' est une constante définie par le protocole pour cibler une inflation spécifique.

4. Limitations & Avertissements

Afin de garantir une transparence totale ("Institutional Grade"), nous listons ici les limites connues de nos modèles :

  • Latence des Données : Les prix affichés peuvent avoir un décalage de 60 secondes par rapport au marché spot (API Caching).
  • Risques "Cygne Noir" : Les simulations de Stress Test (ex: Depeg) s'arrêtent à une déviation de -50%. Elles ne modélisent pas un effondrement total de l'infrastructure (L1 liveness failure).
  • Fiscalité : Les calculs de ROI sont "Bruts de fiscalité". Aucune hypothèse fiscale (Flat Tax, IS) n'est intégrée.
